## Workflow
This project uses the Andromeda planning system:

1. `/andromeda-alpha` — plan architecture for a new project
2. `/andromeda-sigma` — add a new scope to an existing project
3. `/andromeda-beta` — create master development plan from architecture
4. `/andromeda-gamma` — create implementation plan for a specific phase
5. `/setup-project` — configure Claude Code hooks/rules/docs once architecture.md exists
6. `/implement` — execute an implementation plan
7. `/andromeda-help` — full Andromeda documentation

**Typical flow for a new project:**
1. `/andromeda-alpha` → creates `.andromeda/architecture.md` + `.andromeda/project.yaml`
2. `/setup-project` → populates this CLAUDE.md, creates `.claude/rules/` and `.claude/docs/`
3. `/andromeda-beta` → creates masterplan with development phases
4. `/andromeda-gamma` → pick a phase, get implementation plan
5. `/implement` → write code, test, commit, mark phase completed
6. Repeat 4-5 per phase

**Session lifecycle (use at session boundaries):**
- `/new-session` — start of session, shows dashboard + health check + Andromeda status
- `/wrap-session` — end of session, commits work + curates learnings into the right tier + updates handoff
