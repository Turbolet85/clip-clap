# Technology Stack

_Mirror of the Stack and Technologies table in `.andromeda/architecture.md`. Generated by `/setup-project`. Primary source remains architecture.md ‚Äî this file is a convenience reference for on-demand reads._

## Languages & Runtimes
- **Go 1.23+** (forward-compat tested on 1.24) ‚Äî entire application; cross-compilation target, static binary, `log/slog` stdlib. Bumped from 1.22 per security-plan ¬ßDependency Security (CVE-2025-47913 / -47914 / -58181)
- **Python 3 + `pytest` 8.x** ‚Äî integration test runner only (not shipped to users)
- **PowerShell** (`scripts/agent-run.ps1`) ‚Äî verification harness orchestrator

## Core Frameworks & Libraries
- **Systray:** direct Win32 `Shell_NotifyIcon` + `TrackPopupMenuEx` via `golang.org/x/sys/windows`
- **Overlay:** direct Win32 `CreateWindowExW`, `WS_EX_LAYERED`, `UpdateLayeredWindow` via `golang.org/x/sys/windows`
- **Hotkey:** direct Win32 `RegisterHotKey` + `WM_HOTKEY` via `golang.org/x/sys/windows`
- **Clipboard:** direct Win32 `OpenClipboard` / `EmptyClipboard` / `GlobalAlloc` / `SetClipboardData(CF_UNICODETEXT)` via `golang.org/x/sys/windows`
- **Screen capture:** `github.com/kbinani/screenshot` pseudo-version `v0.0.0-20250624051815-089614a94018` (MIT, pure-Go) ‚Äî `CaptureRect` for selected region; upstream does not tag recent commits
- **Toast notification:** `github.com/go-toast/toast` pseudo-version `v0.0.0-20190211030409-01e6764cf0a4` (upstream frozen 2019; `stuartleeks/toast` fork NOT used per security-plan ¬ßDependency Security ‚Äî unmaintained, no reviewed commits)
- **Single-instance guard:** direct Win32 `CreateMutex` via `golang.org/x/sys/windows` ‚Äî mutex name `Local\ClipClapSingleInstance` per security-plan (per-session namespace, not `Global\`)
- **TOML config parser:** `github.com/pelletier/go-toml/v2` v2.2.2 (exact pin; bumped from v2.2.0 per security-plan ¬ßDependency Security)
- **ULID:** `github.com/oklog/ulid/v2` v2.1.0 (explicit `ulid.Monotonic(crypto/rand.Reader, 0)` per security-plan; never `ulid.Make()` ‚Äî non-monotonic)
- **Icon loading:** `ExtractIconW(hInstance, exePath, 0)` ‚Äî index-based extraction from the running `.exe` via `GetModuleFileNameW`; NOT `LoadIconW(resourceID)` (goversioninfo resource-name scheme varies by version)
- **Resource embedding:** `github.com/josephspurrier/goversioninfo` v1.5.0 (tagged release)
- **Win32 UI constants:** most `MOD_*`, `VK_*`, `WM_*`, and procs (`RegisterHotKey`, `Shell_NotifyIconW`, `CreateWindowExW`, `GetMessageW`, etc.) are NOT exported by `x/sys/windows` v0.24.0 ‚Äî defined locally per-subsystem + lazy-loaded via `windows.NewLazySystemDLL(...).NewProc(...)` per architecture.md ¬ß[Win32 API Surface]

## Data Storage
- **None.** TOML config file in `%APPDATA%\clip-clap\config.toml` + in-memory Undo snapshot only. No RDBMS, no SQL, no ORM, no migrations

## Messaging & Events
- **N/A.** Single process, single user. No queue, no event bus, no inter-process messaging beyond the loopback status endpoint

## Observability
- **Logging:** Go stdlib `log/slog` JSONHandler ‚Üí `logs/agent-latest.jsonl`
- **Status endpoint:** Go stdlib `net/http` on `127.0.0.1:27773/status` (loopback-only, `--agent-mode`-gated)
- **No OpenTelemetry, no Prometheus, no distributed tracing** by design

## Development & CI
- **Formatter:** `gofmt` (Go stdlib) ‚Äî wired into Claude Code hooks
- **Linter:** not declared in architecture; recommend `golangci-lint` if installed (not enabled by default)
- **Type checker:** built-in via `go build` / `go vet` (no separate hook needed)
- **CI:** GitHub Actions on `windows-latest` runner ‚Äî `test-unit`, `test-verification`, `build-release`, `publish-release` jobs
- **Coverage:** `go test ./... -cover` target ‚â• 60%

## Build & Distribution
- **Build:** `go build -ldflags="-H windowsgui -s -w" -o clip-clap.exe ./cmd/clip-clap` with `CGO_ENABLED=0` enforced
- **Resource codegen:** `//go:generate goversioninfo -icon=assets/app.ico -manifest=assets/app.manifest`
- **Code signing:** v1.0 unsigned; v1.1+ Azure Trusted Signing via GitHub Actions secret `AZURE_SIGN_THUMBPRINT`
- **Distribution:** GitHub Releases (canonical), Scoop manifest in personal bucket; Winget deferred until install count > ~50

## Verification
- **Profile:** `desktop-windows` (per Andromeda Phase 4 catalog)
- **Driver:** `pywinauto` v0.6.8 (UIA backend, MIT) ‚Äî Python ‚Üî Windows UI Automation bridge
- **Test runner:** `pytest` 8.x
- **Harness:** `scripts/agent-run.ps1 {build|start|status|logs|kill}`
- **Status contract:** `localhost:27773/status` JSON; logs `logs/agent-latest.jsonl` slog JSON

## Third-party services
- **None at runtime.** GitHub (releases, CI), Scoop (personal bucket) are distribution channels, not runtime dependencies

## Rationale

For architectural rationale behind these choices, see `.andromeda/architecture.md` Established Decisions section. Each `[Tag]` in that section explains why a specific technology was chosen over alternatives considered during `/andromeda-alpha`.

## Version updates

When updating a dependency version:
1. Update `.andromeda/architecture.md` Stack table first
2. Update `go.mod` exact pin (and run `go mod tidy`)
3. Re-run `/setup-project` to refresh this file
4. Verify `go test ./... -race` and the integration suite still pass
5. Commit as `chore: bump {module} to {version}`
